# Markdown Formatting

## When to Use

Use this skill when organizing README sections, code blocks, tables, alerts, and links so content renders correctly and stays easy to navigate in GitHub-flavored Markdown.

## Instructions

1. Use a consistent heading hierarchy with ATX headers and no skipped levels.
2. Wrap command examples in fenced code blocks with language hints (`bash`, `json`, `yaml`, etc.).
3. Use tables only for genuinely tabular data; use lists for sequential guidance.
4. Prefer relative links for repository-local docs and assets.
5. Keep lines reasonably short for diff readability and review ergonomics.
6. Verify that badges, anchors, and code fences render as intended in GitHub.
