# Feynman Technique Explainer

## When to Use
After first-principles breakdown, or whenever a concept needs to be explained in genuinely simple terms.

## Instructions

Follow Richard Feynman's method — if you can't explain it simply, you don't understand it well enough.


1. **Start from zero** — Pretend the listener is a curious 12-year-old who has never encountered any of these ideas. No jargon. No "as you probably know." No assumed background
2. **Use everyday analogies** — Connect the concept to things from daily life:
   - Cooking, building with LEGO, playing sports, conversations with friends, how a school works, how water flows, how a library is organized
   - The analogy should make someone say "oh, it's like THAT"
3. **One idea at a time** — Don't pile concepts. Introduce one piece, make sure it's solid, then add the next. Each new idea should build naturally on the last
4. **Catch yourself** — Every time you write a sentence, re-read it and ask:
   - Would a 12-year-old understand every word in this sentence?
   - Am I using a technical term? If yes, replace it or define it in plain words first
   - Is there a simpler way to say this?
5. **Test for hidden complexity** — If any part of the explanation still feels complicated:
   - That's a signal there's another layer to unpack
   - Break that sub-concept down further
   - Keep going until it is genuinely, honestly simple
6. **Confirm understanding** — End by summarizing the entire concept in 2-3 sentences that a kid could repeat back to you. If the summary doesn't work, the explanation isn't done yet
