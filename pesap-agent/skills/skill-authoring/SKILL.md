---
name: skill-authoring
description: Create concise, safe, reusable skills from user-supplied topics or procedures
---

## Use when
- The user asks to create or refine a reusable skill.

## Steps
1. Define scope and boundaries.
2. Write concise instructions.
3. Include trigger conditions.
4. Include output contract.
5. Verify safety constraints.

## Output
- New or updated `skills/<name>/SKILL.md`
- Rationale for when the skill should be used
