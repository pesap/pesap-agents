# Technical Writing

## When to Use

Use this skill when creating or revising README prose, API explanations, setup instructions, or troubleshooting guidance that must be clear for both new and experienced developers.

## Instructions

1. Start each section with the user outcome, then provide only the details needed to achieve it.
2. Prefer concrete nouns and active verbs; remove vague filler and marketing language.
3. Keep terminology consistent with the codebase and CLI/API naming.
4. Use short paragraphs and lists so readers can scan quickly.
5. Validate technical claims against project artifacts before stating them as facts.
6. If uncertainty exists, label assumptions explicitly instead of guessing.
