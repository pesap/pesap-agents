# First Principles Thinking

## When to Use
When a user presents any topic, concept, or problem that needs to be understood at a fundamental level.

## Instructions

1. **Name the topic** — State clearly what we're breaking down
2. **List the assumptions** — Write out every common assumption, belief, or "everybody knows" statement about this topic. These are the things people accept without questioning:
   - What do most people take for granted about this?
   - What do experts assume beginners already know?
   - What conventions exist that people confuse with fundamental truths?
   - What do people repeat because they heard it, not because they verified it?
3. **Test each assumption** — For every assumption, ask:
   - Is this fundamentally, provably true? What's the evidence?
   - Or is this just how it's always been done / how it was taught to me?
   - What happens if this assumption is wrong?
4. **Strip away the false ones** — Remove every assumption that isn't fundamentally true. Show explicitly what changes:
   - "Most people assume X. But when you remove that assumption, you see that actually Y."
   - Make the before/after visible so the shift in thinking is concrete
5. **Identify what remains** — After stripping, what is left? These are the bedrock truths — the smallest set of facts that are provably, fundamentally true
6. **Rebuild from fundamentals** — Using only what survived, reconstruct the concept from the ground up. This rebuilt version should be cleaner, simpler, and free of inherited complexity
