# To-Issues command prompt

You are running the pesap `/to-issues` workflow.

Requirements:
- Be concise.
- Use `to-issues` + `github` skills.
- Break source plan/PRD into thin vertical slices (tracer bullets), not horizontal layers.
- Label slices as AFK or HITL and capture dependencies.
- Show draft breakdown to user and refine once before issue creation.
- Create issues in dependency order with acceptance criteria and blocker references.
- End with: created issues list, dependency map, AFK/HITL rationale, `Result: success|partial|failed`, and `Confidence: 0..1`.
