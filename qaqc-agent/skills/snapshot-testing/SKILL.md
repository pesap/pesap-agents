# Snapshot Testing

## When to Use
When testing serialization output, CLI output, API response shapes, rendered templates, error messages, or any output where writing the expected value by hand is tedious and the "correct" output is best defined as "what it was last time."

## When NOT to Use
- Non-deterministic output (timestamps, random IDs, memory addresses). Filter these first or don't snapshot
- Trivial values you can assert directly (`assert result == 42` is better than a snapshot)
- Highly volatile output that changes every PR (snapshots become rubber-stamp updates)

## Instructions

1. **Install syrupy**:

```bash
uv add --dev syrupy
```

2. **Basic snapshot test** — The `snapshot` fixture is auto-discovered:

```python
def test_user_serialization(snapshot):
    user = User(name="Alice", role="admin")
    assert user.to_dict() == snapshot

def test_cli_help_output(snapshot, capsys):
    main(["--help"])
    captured = capsys.readouterr()
    assert captured.out == snapshot
```

On the first run, syrupy creates a `__snapshots__/` directory with `.ambr` files containing the expected output. Subsequent runs compare against the stored snapshot.

3. **Update snapshots** — When output changes intentionally:

```bash
# Update all snapshots
pytest --snapshot-update

# Update only snapshots for specific tests
pytest --snapshot-update -k test_user_serialization

# Review what changed in git diff after updating
git diff tests/__snapshots__/
```

Always review the diff before committing updated snapshots. Blindly updating defeats the purpose.

4. **Snapshot serialization formats** — syrupy supports multiple formats:

```python
from syrupy.extensions.json import JSONSnapshotExtension
from syrupy.extensions.single_file import SingleFileSnapshotExtension

# JSON snapshots (readable, diffable)
@pytest.fixture
def snapshot_json(snapshot):
    return snapshot.use_extension(JSONSnapshotExtension)

def test_api_response(snapshot_json):
    response = client.get("/users")
    assert response.json() == snapshot_json

# Single file per snapshot (useful for large outputs)
@pytest.fixture
def snapshot_file(snapshot):
    return snapshot.use_extension(SingleFileSnapshotExtension)
```

5. **Filter non-deterministic fields** — Sanitize before snapshotting:

```python
def test_created_user_response(snapshot, client):
    response = client.post("/users", json={"name": "Alice"})
    data = response.json()

    # Replace non-deterministic fields before snapshot comparison
    data["id"] = "<ID>"
    data["created_at"] = "<TIMESTAMP>"

    assert data == snapshot
```

Or use a custom serializer:

```python
from syrupy.filters import props

def test_user_response(snapshot):
    response = create_user("Alice")
    # Exclude volatile fields from the snapshot
    assert response == snapshot(exclude=props("id", "created_at", "updated_at"))
```

6. **Snapshot for parametrized tests** — Each parameter combination gets its own snapshot:

```python
@pytest.mark.parametrize("format", ["json", "yaml", "toml"])
def test_config_export(snapshot, format):
    config = Config(debug=True, port=8080)
    assert config.export(format) == snapshot
```

This creates separate snapshot entries for `test_config_export[json]`, `test_config_export[yaml]`, etc.

7. **Inline snapshots** — For small values, keep the snapshot in the test file:

```python
from syrupy.extensions.amber import AmberSnapshotExtension

def test_error_message(snapshot):
    with pytest.raises(ValidationError) as exc:
        validate({"name": ""})
    assert str(exc.value) == snapshot
```

8. **Clean up unused snapshots** — After deleting or renaming tests:

```bash
# Delete snapshots that no longer match any test
pytest --snapshot-update --snapshot-warn-unused
```

Configure in `pyproject.toml`:

```toml
[tool.pytest.ini_options]
# Warn about unused snapshots on every run
addopts = "--snapshot-warn-unused"
```

9. **Commit snapshots to version control** — Snapshot files (`__snapshots__/`) must be committed. They're the "expected output" and the diff in PRs shows exactly what changed.

## Project Structure

```
tests/
├── __snapshots__/
│   └── test_serialization.ambr    # auto-generated by syrupy
├── unit/
│   ├── __snapshots__/
│   │   └── test_models.ambr
│   └── test_models.py
└── conftest.py
```

## Anti-Patterns to Fix
- Snapshots with timestamps, UUIDs, or other non-deterministic data (filter them)
- `--snapshot-update` run without reviewing the diff (defeats the whole point)
- Huge snapshots that nobody reads during review (break into smaller tests or use exclude filters)
- Snapshots for trivial assertions (`assert x == 5` doesn't need a snapshot)
